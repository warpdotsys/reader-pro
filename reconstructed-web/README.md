# Reconstructed web notes

No source maps were found in `BOOT-INF/classes/web` for `reader-pro-3.2.14.jar`, so this directory does **not** claim to contain authoritative rebuildable Vue source.

The authoritative shipped frontend remains in `recovered-assets/web/`, extracted byte-for-byte from the JAR. This directory contains audit aids derived from those bundles:

- `bundle-index.json` records bundle names, sizes, and SHA-256 hashes.
- `routes-and-api-strings.txt` lists best-effort route/API-like strings found in JavaScript bundles.

Use these files to guide future source reconstruction, not as a replacement for the shipped assets.
